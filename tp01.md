# Productos o servicios que utilizan Machine Learning

## 1. Duolingo – Personalización del aprendizaje

**Nombre del producto/servicio**

Duolingo

**Qué problema resuelve con ML**

En una plataforma para aprender idiomas cada persona aprende a un ritmo distinto. Algunas personas necesitan repetir más ejercicios mientras que otras avanzan más rápido.

Duolingo usa machine learning para ajustar las lecciones según el progreso del usuario, mostrando ejercicios que se adapten a lo que ya sabe o a los errores que suele cometer.

**Tipo de Machine Learning**

Aprendizaje supervisado.

Se puede considerar supervisado porque el sistema aprende a partir de datos donde ya se conoce el resultado, por ejemplo si un ejercicio fue respondido correctamente o incorrectamente. Con esos datos el modelo puede estimar qué tan difícil será un ejercicio para el usuario.

**Datos que probablemente necesita**

- Respuestas correctas e incorrectas  
- Tiempo que tarda el usuario en responder  
- Historial de ejercicios realizados  
- Nivel actual dentro del idioma

## 2. Shazam – Identificación de canciones

**Nombre del producto/servicio**

Shazam

**Qué problema resuelve con ML**

Muchas veces escuchamos una canción pero no sabemos cuál es. Shazam permite identificarla grabando unos segundos de audio desde el celular.

La aplicación analiza el fragmento de sonido y lo compara con una base de datos de canciones para encontrar coincidencias.

**Tipo de Machine Learning**

Aprendizaje supervisado.

Se puede considerar supervisado porque el sistema se entrena con canciones que ya están identificadas. El modelo aprende patrones del audio de cada canción y luego puede compararlos con el audio grabado por el usuario.

**Datos que probablemente necesita**

- Grabaciones de canciones  
- Características del sonido (frecuencias, ritmo, intensidad)  
- Base de datos de canciones registradas  
- Fragmentos de audio capturados por los usuarios

## 3. Notion AI – Asistencia para escritura

**Nombre del producto/servicio**

Notion AI

**Qué problema resuelve con ML**

Dentro de Notion se pueden escribir notas, documentos o ideas. El asistente de IA ayuda a generar texto, resumir información o mejorar la redacción de un documento.

Esto sirve para organizar información más rápido o para tener una base de texto cuando se empieza a escribir algo.

**Tipo de Machine Learning**

Aprendizaje supervisado.

Se considera supervisado porque los modelos se entrenan con grandes cantidades de textos donde el sistema aprende patrones del lenguaje. Con ese entrenamiento puede generar texto o resumir contenido.

**Datos que probablemente necesita**

- Grandes colecciones de textos  
- Frases escritas por usuarios  
- Documentos que se quieren resumir o modificar

## 4. Strava – Análisis de actividad deportiva

**Nombre del producto/servicio**

Strava

**Qué problema resuelve con ML**

Strava registra actividades deportivas como correr o andar en bicicleta usando GPS. Con esos datos puede analizar el rendimiento del usuario y detectar patrones en sus entrenamientos.

También permite identificar rutas populares que usan muchos deportistas.

**Tipo de Machine Learning**

Aprendizaje no supervisado.

Se puede considerar no supervisado porque el sistema analiza muchos datos de actividades para encontrar patrones o agrupaciones, por ejemplo rutas similares o comportamientos de entrenamiento, sin que esos datos estén previamente etiquetados.

**Datos que probablemente necesita**

- Datos de GPS de las actividades  
- Distancia recorrida  
- Velocidad promedio  
- Tiempo de actividad  
- Historial de entrenamientos  

---

## 5. Ualá – Detección de fraude en transacciones

**Nombre del producto/servicio**

Ualá

**Qué problema resuelve con ML**

Las aplicaciones financieras necesitan detectar compras sospechosas para evitar fraudes.

Ualá utiliza modelos de machine learning para analizar las transacciones y detectar comportamientos que no coinciden con el uso normal de la cuenta.

**Tipo de Machine Learning**

Aprendizaje supervisado.

Se considera supervisado porque el sistema puede entrenarse con transacciones que ya fueron clasificadas como legítimas o fraudulentas. Con esos ejemplos aprende patrones que luego usa para evaluar nuevas operaciones.

**Datos que probablemente necesita**

- Historial de transacciones  
- Ubicación desde donde se realiza el pago  
- Dispositivo utilizado  
- Frecuencia de compras  
- Comercios donde compra el usuario normalmente
